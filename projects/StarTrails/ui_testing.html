User-Interface Testing

After implementing our major functionality and the majority of our user interface, we conducted User-Interface tests to see which parts of our implementation worked well and what aspects could or should be changed to create a better end product.

Methods

Our User-Interface testing methodology comprised of:

  • Recruitment of Testers: We recruited a diverse group of six participants, each from different backgrounds and relationships with stargazing.
  • Scenarios & Tasks: We designed realistic scenarios and tasks to evaluate the app's core features and usability.
  • Recording & Observation: We recorded screen activity using screen recording software and took notes on participant behavior and feedback.
  • Feedback Gathering: Conducted post-test questionnaires and interviews to collect detailed feedback on user satisfaction and suggestions for improvement.

Summary

Based on the user testing sessions conducted with six participants, here is a summary of the findings:

  • Number of Testing Sessions: 6
  • User Pairs: None (due to inability to recruit participants willing to work in pairs)
  • Common Positive Feedback:
    • Participants found the app to be relatively intuitive and easy to use, rating it an average of 8 out of 10.
    • Users appreciated features like nearby sites, filter functionality, and the ability to access weather information.
    • Distinction between visited and favorite sites was well-received.
    • Participants liked the level of detail provided on site detail pages, including weather integration and light pollution data.
    • Most users found functionalities like leaving comments and setting notifications to be straightforward.
  • Common Issues Encountered:
    • Navigation and Clunkiness: Some participants found the navigation clunky, with delays when clicking on items, and getting lost in the navigation at times.
    • UI Design Confusion: Confusion with certain UI elements, such as icons for amenities, and the layout of the site view screen.
    • Keyboard Overlapping: Issues with the keyboard blocking important UI elements, particularly when leaving comments or adding photos.
    • Functionality Bugs: Bugs with functionalities like deleting comments, saving sites, and adding photos.
    • Incomplete Features: Missing or incomplete features, such as sorting by weather and clearer distinction for favorited sites on the map.

Resulting Changes

Based on the feedback collected from the user testing sessions, we implemented the following changes:

  • Navigation Improvements: Streamlined navigation and reduced delays by optimizing the app's performance. Provided clearer feedback on navigation actions.
  • UI Design Enhancements: Clarified UI elements, such as icons for amenities, and redesigned complex screens like the site view screen for better usability.
  • Keyboard Management: Prevented the keyboard from overlapping important UI elements, such as comment submission buttons.
  • Bug Fixes: Addressed functionality bugs to ensure a smooth user experience. Thoroughly tested all features to identify and resolve any existing issues.
  • Feature Completion: Prioritized completing missing or incomplete features based on user feedback, such as adding sorting options and improving visibility for favorited sites on the map.
